# 用户文档：串口通讯使用说明

## 1. 文档目的

本文面向上位机开发者、PLC/单片机/工控主机集成开发者，说明如何通过串口与 8 路颜色对比传感器通讯，读取设备数据并解析返回字段。

本文只描述用户需要关心的通讯方法和字段含义，不涉及固件内部实现细节。

## 2. 产品串口能做什么

通过串口，主机可以完成以下操作：

- 检测设备是否在线。
- 按需查询 8 路传感器数据。
- 查询设备基本信息。
- 开启或停止实时数据流。
- 识别设备返回的错误响应。

## 3. 串口参数

| 项目 | 参数 |
| --- | --- |
| 通讯方式 | UART 串口 |
| 波特率 | `115200` |
| 数据位 | `8 bit` |
| 校验位 | 无校验 |
| 停止位 | `1 bit` |
| 硬件流控 | 无 |
| 常用表示 | `115200 8N1` |

## 4. 通讯方式概览

串口通讯包含三种形式：

1. **PING 通讯**：主机发送 `CC`，设备返回 `CC`，用于快速确认连接。
2. **标准请求/响应帧**：主机发送 `AA ... BB` 格式的请求帧，设备返回同样结构的响应帧。
3. **实时数据流**：主机发送实时流控制字节后，设备按周期输出固定长度数据帧。

如果只需要偶尔读取一次数据，建议使用“传感器数据查询”。如果需要连续刷新数据，建议使用“实时数据流”。

## 5. PING 通讯

PING 用于确认设备在线和串口链路可用。

### 5.1 请求

```text
CC
```

### 5.2 响应

```text
CC
```

### 5.3 说明

- PING 是单字节通讯，不使用标准帧格式。
- 主机发送 `0xCC` 后，设备正常在线时返回 `0xCC`。

## 6. 标准帧格式

标准帧用于请求/响应类命令。请求帧和响应帧结构相同。

```text
AA Command Length Payload CRC_L CRC_H BB
```

| 顺序 | 字段 | 长度 | 说明 |
| --- | --- | --- | --- |
| 1 | 帧头 | 1B | 固定为 `0xAA`，表示标准帧开始 |
| 2 | Command | 1B | 命令码，表示当前帧的业务类型 |
| 3 | Length | 1B | Payload 的字节数 |
| 4 | Payload | N 字节 | 负载数据，长度由 Length 指定 |
| 5 | CRC_L | 1B | CRC-16 低字节 |
| 6 | CRC_H | 1B | CRC-16 高字节 |
| 7 | 帧尾 | 1B | 固定为 `0xBB`，表示标准帧结束 |

### 6.1 Length 字段

`Length` 只表示 `Payload` 的长度，不包含帧头、命令码、Length 本身、CRC 和帧尾。

例如：

```text
AA 01 00 C0 21 BB
```

含义：

| 字节 | 含义 |
| --- | --- |
| `AA` | 帧头 |
| `01` | 命令码：传感器数据查询请求 |
| `00` | Payload 长度为 0 |
| `C0 21` | CRC-16，低字节在前 |
| `BB` | 帧尾 |

## 7. CRC-16 校验

标准帧使用 CRC-16 校验。

| 项目 | 说明 |
| --- | --- |
| 算法 | CRC-16/MODBUS |
| 初始值 | `0xFFFF` |
| 多项式 | `0xA001` |
| 输出字节序 | 低字节在前，高字节在后 |
| 计算范围 | `Command + Length + Payload` |

CRC 不包含：

- 帧头 `0xAA`
- CRC 字段本身
- 帧尾 `0xBB`

## 8. 常用快速开始

### 8.1 检查设备是否在线

主机发送：

```text
CC
```

设备返回：

```text
CC
```

### 8.2 查询一次传感器数据

主机发送标准帧：

```text
AA 01 00 C0 21 BB
```

设备返回命令码为 `0x81` 的标准帧，Payload 长度为 20 字节。

### 8.3 开启实时数据流

主机发送：

```text
DD 0A
```

表示开启实时数据流，其中 `Interval` 为 1 字节输出间隔，单位是毫秒；`0x0A` 表示每 10 ms 输出一次。

### 8.4 停止实时数据流

主机发送：

```text
DD 00
```

表示停止实时数据流。

## 9. 传感器数据查询

传感器数据查询用于读取 8 路数字量、模拟量、归一化值和偏移量。

### 9.1 请求

| 项目 | 值 |
| --- | --- |
| Command | `0x01` |
| Length | `0x00` |
| Payload | 无 |

完整请求示例：

```text
AA 01 00 C0 21 BB
```

### 9.2 响应

| 项目 | 值 |
| --- | --- |
| Command | `0x81` |
| Length | `0x14`，即 20 字节 |
| Payload | 传感器数据 |

Payload 字段定义：

| 偏移 | 字段 | 长度 | 说明 |
| --- | --- | --- | --- |
| 0 | Status | 1B | 状态字节，当前 `0x00` 表示成功 |
| 1 | Digital | 1B | 8 路数字量输出位图 |
| 2-9 | Analog | 8B | 8 路模拟量，每路 1 字节 |
| 10-17 | Normal | 8B | 8 路归一化值，每路 1 字节 |
| 18-19 | Offset | 2B | 偏移量，有符号 16 位整数，低字节在前 |

### 9.3 Digital 字段

`Digital` 是 8 路数字量输出位图。

| bit | 通道 |
| --- | --- |
| bit0 | 通道 0 |
| bit1 | 通道 1 |
| bit2 | 通道 2 |
| bit3 | 通道 3 |
| bit4 | 通道 4 |
| bit5 | 通道 5 |
| bit6 | 通道 6 |
| bit7 | 通道 7 |

示例：

```text
Digital = 05
```

`0x05` 的二进制为 `0000 0101`，表示通道 0 和通道 2 输出为 1。

### 9.4 Analog 字段

`Analog` 为 8 字节，依次表示通道 0 到通道 7 的模拟量。

| 字节位置 | 通道 |
| --- | --- |
| Analog[0] | 通道 0 |
| Analog[1] | 通道 1 |
| Analog[2] | 通道 2 |
| Analog[3] | 通道 3 |
| Analog[4] | 通道 4 |
| Analog[5] | 通道 5 |
| Analog[6] | 通道 6 |
| Analog[7] | 通道 7 |

### 9.5 Normal 字段

`Normal` 为 8 字节，依次表示通道 0 到通道 7 的归一化值。

| 字节位置 | 通道 |
| --- | --- |
| Normal[0] | 通道 0 |
| Normal[1] | 通道 1 |
| Normal[2] | 通道 2 |
| Normal[3] | 通道 3 |
| Normal[4] | 通道 4 |
| Normal[5] | 通道 5 |
| Normal[6] | 通道 6 |
| Normal[7] | 通道 7 |

### 9.6 Offset 字段

`Offset` 是 2 字节数据。解析时先将两个字节按小端序组合成一个 16 位值，然后再将该 16 位值按 signed int16 解释。

示例：

| 原始字节 | 解析结果 |
| --- | --- |
| `01 00` | `1` |
| `FF FF` | `-1` |
| `00 80` | `-32768` |

## 10. 设备信息查询

设备信息查询用于读取硬件版本、设备运行时间和当前运行模式。

### 10.1 请求

| 项目 | 值 |
| --- | --- |
| Command | `0x04` |
| Length | `0x00` |
| Payload | 无 |

完整请求示例：

```text
AA 04 00 03 70 BB
```

其中 `04 00` 的 CRC-16/MODBUS 计算结果为 `0x7003`，发送时按低字节在前、高字节在后的顺序写入，因此 CRC 字节为 `03 70`。

### 10.2 响应

| 项目 | 值 |
| --- | --- |
| Command | `0x84` |
| Length | `0x06`，即 6 字节 |
| Payload | 设备信息 |

Payload 字段定义：

| 偏移 | 字段 | 长度 | 说明 |
| --- | --- | --- | --- |
| 0 | HardwareVersion | 1B | 硬件版本号 |
| 1-4 | RuntimeSeconds | 4B | 设备运行时间，单位秒，低字节在前 |
| 5 | RunMode | 1B | 当前运行模式 |

`RunMode` 定义：

| 值 | 含义 |
| --- | --- |
| `0x00` | 普通运行模式 |

## 11. 实时数据流

实时数据流用于让设备按周期主动输出传感器数据。

实时数据流不使用标准帧格式，而是使用固定长度数据帧。

### 11.1 开启或更新实时数据流

主机发送：

```text
DD Interval
```

| 字节 | 说明 |
| --- | --- |
| `DD` | 实时流控制头 |
| `Interval` | 1 字节输出间隔，单位为毫秒；非零表示开启或更新实时输出周期，`00` 表示停止流输出 |

示例：

```text
DD 0A
```

表示开启实时数据流，输出间隔为 10 ms。

### 11.2 停止实时数据流

主机发送：

```text
DD 00
```

`Interval` 为 `0x00` 时，设备停止实时数据流。

### 11.3 实时数据帧格式

设备输出固定长度 21 字节数据帧：

```text
DD Digital Analog[0] ... Analog[7] Normal[0] ... Normal[7] Offset_L Offset_H EE
```

| 偏移 | 字段 | 长度 | 说明 |
| --- | --- | --- | --- |
| 0 | 帧头 | 1B | 固定为 `0xDD` |
| 1 | Digital | 1B | 8 路数字量输出位图 |
| 2-9 | Analog | 8B | 8 路模拟量，每路 1 字节 |
| 10-17 | Normal | 8B | 8 路归一化值，每路 1 字节 |
| 18-19 | Offset | 2B | 偏移量，有符号 16 位整数，低字节在前 |
| 20 | 帧尾 | 1B | 固定为 `0xEE` |

### 11.4 实时数据流解析注意事项

- 实时数据帧长度固定为 21 字节。
- 实时数据帧没有 Command、Length 和 CRC 字段。
- 解析实时流时，应查找帧头 `0xDD` 和帧尾 `0xEE`，并确认长度为 21 字节。
- 实时流中的 `Digital`、`Analog`、`Normal`、`Offset` 字段含义与传感器数据查询响应中的同名字段一致。

## 12. 错误响应

当设备收到不支持的命令、长度错误或参数错误时，可能返回错误响应。

错误响应使用标准帧格式。

| 项目 | 值 |
| --- | --- |
| Command | `0x7F` |
| Length | `0x02` |
| Payload | 原始命令码 + 错误码 |

Payload 字段定义：

| 偏移 | 字段 | 长度 | 说明 |
| --- | --- | --- | --- |
| 0 | OriginCommand | 1B | 触发错误的原始命令码 |
| 1 | ErrorCode | 1B | 错误码 |

错误码定义：

| 错误码 | 含义 |
| --- | --- |
| `0x01` | 不支持的命令 |
| `0x02` | 数据长度错误 |
| `0x03` | 参数值错误 |
| `0x04` | 执行失败 |

示例：

```text
Command = 7F
Payload = 99 01
```

表示命令 `0x99` 不被支持。

## 13. 字段与字节序说明

### 13.1 十六进制表示

本文中的字节均使用十六进制表示。例如：

```text
AA 01 00 C0 21 BB
```

表示依次发送 6 个字节：`0xAA`、`0x01`、`0x00`、`0xC0`、`0x21`、`0xBB`。

### 13.2 多字节字段

多字节字段采用小端序，即低字节在前，高字节在后。

例如 `Offset` 原始字节为：

```text
34 12
```

解析为：

```text
0x1234
```

### 13.3 标准帧解析建议

解析标准帧时建议按以下顺序处理：

1. 查找帧头 `0xAA`。
2. 读取 `Command`。
3. 读取 `Length`。
4. 根据 `Length` 读取指定长度的 `Payload`。
5. 读取 2 字节 CRC。
6. 确认帧尾为 `0xBB`。
7. 对 `Command + Length + Payload` 计算 CRC，并与收到的 CRC 比较。
8. CRC 正确后，再按命令码解析 Payload。

### 13.4 实时流解析建议

解析实时数据流时建议按以下顺序处理：

1. 查找帧头 `0xDD`。
2. 连续读取 21 字节。
3. 确认第 21 字节为帧尾 `0xEE`。
4. 按固定偏移解析 `Digital`、`Analog`、`Normal` 和 `Offset`。

## 14. 常见问题

### 14.1 为什么发送 `CC` 后没有返回？

请检查：

- 串口号是否正确。
- 波特率是否为 `115200`。
- 数据格式是否为 `8N1`。
- TX/RX 是否接反。
- 设备是否正常供电。

### 14.2 为什么标准帧没有响应？

请检查：

- 帧头是否为 `AA`。
- 帧尾是否为 `BB`。
- Length 是否等于 Payload 实际长度。
- CRC 是否按 `Command + Length + Payload` 计算。
- CRC 是否低字节在前。

### 14.3 什么时候使用查询，什么时候使用实时流？

- 偶尔读取一次数据：使用传感器数据查询。
- 需要连续显示或绘图：使用实时数据流。
- 需要确认设备是否连接：使用 PING。
